Get LVGL Pro Editor running on your system with installation steps tailored to your operating system.

## Windows

1. Go to [pro.lvgl.io/#download](https://pro.lvgl.io/#download)
2. Click the **Download** button and save the installer to your computer
3. Run the installer and follow the prompts to complete the installation

## macOS

1. Download the appropriate version from [pro.lvgl.io/#download](https://pro.lvgl.io/#download):
   - **ARM** for Apple Silicon (M1/M2/M3/M4)
   - **AMD** for Intel-based Macs
2. Drag the application to your **Applications** folder in Finder
3. Launch the Editor from Applications or Spotlight

## Linux

1. Download the AppImage from [pro.lvgl.io/#download](https://pro.lvgl.io/#download)
2. Make it executable:
   ```bash
   chmod +x LVGL_Pro_Editor.AppImage
   ```
3. Run the application:
   ```bash
   ./LVGL_Pro_Editor.AppImage
   ```

### Troubleshooting

**Issue:** Double-clicking the `.AppImage` file doesn't launch the application, or you see a GTK error:

```bash
(process:3943): Gtk-ERROR **: 11:45:01.301: GTK 2/3 symbols detected.
Using GTK 2/3 and GTK 4 in the same process is not supported
Trace/breakpoint trap (core dumped)
```

**Solution:** Run the AppImage from the terminal with the GTK version specified:

```bash
./LVGL_Pro_Editor.AppImage --gtk-version=3
```

## Compiling C code

Component and screen XMLs run directly from XML, so **most projects work without any extra setup**. The `preview-bin` folder in your project contains the compiled C code — LVGL plus some glue code — used for the live preview. If it isn't there, the editor still works and runs your components and screens from pure XML.

Compilation is only needed when your project also uses **widgets**, or you add **custom C code** in the init functions.

To compile, click the **hammer icon** in the editor. The first time, the editor automatically downloads and installs **EMSDK 4.0.6**, which is used to compile the C code to run in the preview. For this to work, a few tools must already be installed:

| OS | Install |
| --- | --- |
| **Windows** | [Git](https://git-scm.com/download/win) and [Ninja](https://github.com/ninja-build/ninja/releases) |
| **macOS** | Git and Make — both included in the Xcode Command Line Tools: `xcode-select --install` |
| **Linux** | Git and Make — e.g. `sudo apt install git make` (Debian/Ubuntu/Mint), `sudo dnf install git make` (Fedora), or `sudo pacman -S git make` (Arch/Manjaro) |

If a tool is missing, the editor shows a message such as *"Missing Dependency: Emsdk 4.0.6 required for compilation."* Install the listed tools and click the hammer icon again.
