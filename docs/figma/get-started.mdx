---
title: Get Started
description: Install LVGL Flow for Figma, connect it to LVGL Pro Editor, and export your first screen.
---

Design your interface in Figma the way you always have. **LVGL Flow for Figma** reads your
file (screens, components, colors, fonts, spacing) and turns it into
[LVGL XML](../syntax/introduction) files you can open, run, and refine in
[LVGL Pro Editor](../editor). You don't write any code to get started.

There are two halves. The **plugin** lives in Figma and reads your design. **The service**
lives inside LVGL Pro Editor and writes the LVGL files to your project. They talk to each
other on your own machine.

## Before you start

- **The Figma desktop app.** You can browse and use the plugin in the browser, but the connection
  to the Editor is most reliable from the desktop app.
- **LVGL Pro Editor.** The service that writes your files ships inside it. Don't have it
  yet? You can still install the plugin and explore, and set up the Editor when you're ready
  to write files.

<Callout type="success">
  **LVGL Flow for Figma** workflow requires LVGL Pro Editor v2.0.0+ to work.
</Callout>

## Step by step

<Steps>

<Step>

### Install the plugin

Add [**LVGL Flow for Figma**](https://www.figma.com/community/plugin/1362005814860504095/lvgl-flow-for-figma)
from Figma's Community, the same way you'd add any plugin. It stays with your account, so
it's available in every file you open on any Figma
plan.

<Figure
  src="/_static/images/figma/install.png"
  alt="The LVGL Flow for Figma listing in the Figma Community"
  caption="Find LVGL Flow for Figma in the Community and add it once."
/>

Open the file you want to bring across, then launch **LVGL Flow** from your Plugins menu.
The plugin opens and immediately scans your file, starting with your tokens, then components, and
lastly screens.

</Step>

<Step>

### Connect to the Editor

The service is already built into LVGL Pro Editor. In the
Editor, click the **Figma button** in the toolbar to open the **LVGL Figma Flow** panel,
then click **Start Service**.

<Figure
  src="/_static/images/figma/editor-connection.png"
  alt="The LVGL Figma Flow control panel with a Start Service button"
  width={500}
  height={500}
  caption="Click Start Service. The service comes up and the plugin connects automatically."
/>

Back in Figma, the plugin connects on its own: the status dot beside your project name
turns green and it shows the repo, branch, and path it's working in.

<Callout type="success">
  Turn on **Start service when the IDE opens** in the panel's Settings and it's running
  every time the Editor launches.
</Callout>

</Step>

<Step>

### Set your display size

Open the **Project** view in the plugin and set your **display size**, this is the width and
height, in pixels, of the screen you're designing for. This is the one setting the plugin
can't guess, and **exports stay blocked until it's filled in**.

<Figure
  src="/_static/images/figma/project-mode.png"
  alt="The Project tab in LVGL Flow, showing the display-size canvas"
  width={350}
  height={620}
  caption="Set your display size once per file. The optional build settings below can wait."
/>

The build settings below the display size are optional and their defaults work most of the
time. See [Project settings](./project-settings) when you want to fine-tune fonts and
images.

</Step>

<Step>

### Annotate your widgets (optional)

A purely visual screen exports fully with no annotation but a [widget](../built_in_widgets) (a slider, switch, or
label) isn't auto-detected. When a layer is really a control or shows live data, open
**Annotate mode**, select the layer, and tell the plugin what it is.

<Figure
  src="/_static/images/figma/annotate-mode.png"
  alt="Annotate mode with a layer selected"
  width={350}
  height={620}
  caption="Select a layer and confirm the plugin's guess about what it is."
/>

This is also where you point a live layer at a [subject](./subjects) so it updates at
runtime. Annotation is **optional for visual content, required for widgets and
interactivity**. See [Annotating](./annotating) for the details.

</Step>

<Step>

### Export your first screen

Open **Export mode**, pick a screen on the canvas, and choose **Selected Screens**. The
plugin sends it to the service along with only the components and tokens it actually uses.

<Figure
  src="/_static/images/figma/export-mode.png"
  alt="Export mode showing the Design Tokens and Screens sections"
  width={350}
  height={620}
  caption="Export a screen, all your components, or a whole prototype flow. Each row exports on its own."
/>

You can also send **All Globals** (your [styles](../syntax/styles) and tokens), **All
Components**, or a whole prototype **Flow**. See [Exporting](./exporting) for what each
action sends.

</Step>

<Step>

### Find your files

The service writes the LVGL XML into your project folder: a `project.xml` root, a
`globals.xml` of tokens, one folder per [component](../syntax/components), and one file per
[screen](../syntax/screens). Open them in the Editor to run and refine them.

You can come back and export again any time — a developer's hand edits stay untouched. See
[Output structure](./output-structure) for the full layout and [Updating](./updating) for
how re-exports and file ownership work.

</Step>

</Steps>

## What converts on its own, and what needs a nudge

LVGL Flow reads the **structure** in your file, not just the pixels — so the more you lean
on Figma's building blocks, the cleaner the LVGL output.

- **Variables and styles** become design **tokens** in `globals.xml`.
- **Components** come across as reusable [LVGL components](../syntax/components).
- **Variants** fold into a widget's [states](./widget-properties#states) but
  subject-driven looks need your annotation.
- **Auto layout** maps directly onto LVGL's own [flex layout](../syntax/view), so
  responsive rows and columns translate instead of freezing into fixed positions.

In most cases, only **widgets and data bindings** need you to step in and annotate.
